# Trippin AI Privacy Policy

Last updated: TODO

This Privacy Policy is a project policy draft for Trippin AI. It is not legal advice.

## Overview

Trippin AI is a Discord community assistant for beta testing, incident capture, workflow logs, and project feedback.

This policy explains what the bot may process, what it does not intentionally collect, and how beta data may be used.

## Information the Bot May Process

Depending on how users interact with Trippin AI, the bot may process:

- Discord username or display name
- Discord user ID only if technically needed for bot operation
- Server, channel, thread, and message context needed for bot operation
- Beta feedback text
- Incident capture text
- Command usage metadata
- Bot response status or error metadata needed for troubleshooting

## Information the Bot Does Not Intentionally Collect

Trippin AI does not intentionally collect:

- Passwords
- Tokens
- Wallet keys or private keys
- Payment card data
- Government identification
- Intentionally sensitive personal data

Users should not post secrets, passwords, tokens, wallet/private keys, private personal information, private documents, or sensitive data in Discord channels or bot commands.

## How Data May Be Used

Trippin AI may use processed data to:

- Operate bot features
- Improve the beta workflow
- Create incident/build logs
- Track beta feedback
- Troubleshoot errors
- Understand confusing or broken user flows

Bot output may be imperfect and should be reviewed by humans.

## Data Storage

During beta, local or development logs may exist to support testing and troubleshooting.

Trippin AI does not sell user data.

## Data Sharing

Trippin AI does not sell user data.

Content posted in Discord may be visible to server admins, moderators, and other users depending on Discord channel permissions.

Data may be reviewed by project operators or server admins to operate the bot, moderate the server, improve the beta, and troubleshoot issues.

## User Safety

Users should not post secrets, passwords, tokens, wallet/private keys, private personal information, personal documents, or sensitive data.

If sensitive information is accidentally posted, users should delete the message if possible and rotate or revoke the exposed secret through the relevant provider.
